# Sixteen - Sprint Planning Meeting

> Documento operativo per sprint planning. Tema.
> Preparazione stimata: 68%.

## Obiettivo sprint

Spostare **Sixteen** in avanti sui gap P0/P1 piu' vicini al valore, evitando lavoro decorativo o non verificabile.

## Input richiesti

- [PRD](prd.md)
- [Product Roadmap](product-roadmap.md)
- issue e discussion aperte
- quality gates del componente

## Proposta agenda

1. Stato attuale e percentuali
2. Gap bloccanti
3. Decisioni di scope
4. Task candidati per sprint
5. Rischi e dipendenze
6. Definizione di done

## Candidate stories

- chiusura gap P0
- aggiornamento test e smoke test
- riallineamento docs e backlog

## Definizione di done

- codice aderente al requisito
- test pertinenti verdi
- documentazione aggiornata
- issue/discussion allineate

## Retro da pianificare

- cosa ha creato attrito nello sprint precedente
- quali documenti erano fuori sync
- quali automatismi conviene aggiungere
