## 1. Theme Detection

### Algoritmo di Rilevamento Tema

```php
// 1. Leggi APP_URL da laravel/.env
$appUrl = env('APP_URL', 'http://fixcity.local');

// 2. Rimuovi protocollo (http:// o https://)
$domain = str_replace(['http://', 'https://'], '', $appUrl);
// fixcity.local

// 3. Rimuovi www
$domain = str_replace('www.', '', $domain);
// fixcity.local

// 4. Explode da "." e inverti array
$parts = array_reverse(explode('.', $domain));
// ['local', 'fixcity']

// 5. Unisci con "/"
$configPath = implode('/', $parts);
// "local/fixcity"

// 6. Leggi configurazione
$pubTheme = config('xra.pub_theme');
// Da: base_path('config/local/fixcity/xra.php')
// Risultato: "Sixteen"
```

**File di configurazione:**
```php
// config/local/fixcity/xra.php
return [
    'pub_theme' => 'Sixteen',  // ← Nome del tema
    'register_pub_theme' => true,
];
```

**Perché:**
- ✅ Il tema è dinamico e configurabile
- ✅ Puoi cambiare tema (Sixteen → Zero → Seventeen) modificando solo la config
- ✅ Il namespace `pub_theme` è agnostico e portabile

## 2. Vite Build System

### 🔴 Errore Comune

```blade
{{-- SBAGLIATO: Cerca in public_html/build/manifest.json --}}
@vite(['resources/css/app.css', 'resources/js/app.js'])
```

### ✅ Soluzione Corretta

```blade
{{-- CORRETTO: Cerca in public_html/themes/Sixteen/build/manifest.json --}}
@vite(['resources/css/app.css', 'resources/js/app.js'], 'themes/Sixteen')
```

### Configurazione Vite

**File:** `laravel/Themes/Sixteen/vite.config.js`

```javascript
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
  build: {
    outDir: './public',  // Genera in themes/Sixteen/public/build/
    manifest: true,       // Crea manifest.json
  },
  plugins: [
    laravel({
      input: ['resources/css/app.css', 'resources/js/app.js'],
      refresh: true,
    }),
  ],
});
```

**File:** `laravel/Themes/Sixteen/package.json`

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "copy": "cp -rv ./public/* ../../../public_html/themes/Sixteen/"
  }
}
```

### 🐛 Risoluzione Errori Vite Manifest

```bash
# Se vedi: Vite manifest not found at: public_html/themes/Sixteen/manifest.json
cd laravel/Themes/Sixteen
composer update -W
npm install
npm run build
npm run copy
```

**Perché funziona:**
1. `outDir: './public'` → Vite genera in `themes/Sixteen/public/build/manifest.json`
2. `npm run copy` → Sincronizza con `public_html/themes/Sixteen/`
3. `@vite([...], 'themes/Sixteen')` → Cerca nel percorso corretto

## 3. Page Routing con Folio + Volt

### Template Universale [slug].blade.php

**File:** `Themes/Sixteen/resources/views/pages/tests/[slug].blade.php`

```blade
<?php

declare(strict_types=1);

use function Laravel\Folio\middleware;
use function Laravel\Folio\name;
use Livewire\Volt\Component;
use Modules\Cms\Http\Middleware\PageSlugMiddleware;

name('tests.view');
middleware(PageSlugMiddleware::class);

new class extends Component {
    public string $slug = '';
    public string $pageSlug = '';

    /** @var array<string, mixed> */
    public array $data = [];

    public function mount(string $slug): void
    {
        $this->slug = $slug;
        $this->pageSlug = 'tests.'.$slug;
        $this->data = [
            'slug' => $slug
        ];
    }
};

?>

<x-layouts.app>
    @volt('tests.view')
        <div>
            <x-page side="content" :slug="$pageSlug" :data="$data" />
        </div>
    @endvolt
</x-layouts.app>
```

### Pagina Index

**File:** `Themes/Sixteen/resources/views/pages/tests/index.blade.php`

```blade
<?php

declare(strict_types=1);

use function Laravel\Folio\middleware;
use function Laravel\Folio\name;
use Livewire\Volt\Component;
use Modules\Cms\Http\Middleware\PageSlugMiddleware;

    /** @var array<string, mixed> */

        $this->pageSlug = 'tests.index';

?>

{{-- ❌ SBAGLIATO: File specifici --}}

{{-- ❌ SBAGLIATO: Layout diversi --}}

<x-layouts.app>
```

**Perché:**
- ✅ **DRY:** Unico file per tutte le pagine
- ✅ **KISS:** Logica semplice e focalizzata
- ✅ **Manutenibilità:** Modifica una volta, applica a tutte

## 5. Header & Footer Sections

### Header come Section

```blade
{{-- ✅ CORRETTO --}}
<x-section slug="header" />

{{-- Se serve variante con parametri --}}
<x-section slug="header" :data="$headerData" />
```

### Footer come Section

```blade
{{-- ✅ CORRETTO: Footer standard --}}
<x-section slug="footer" />

{{-- ✅ CORRETTO: Footer slim --}}
<x-section slug="footer" tpl="slim" />
```

### ❌ ERRORI DA EVITARE

```blade
{{-- MAI scrivere HTML dell'header direttamente nel layout --}}
<header class="it-header-wrapper">...</header>

{{-- MAI usare @include per pagine dinamiche --}}
@includeIf('pub_theme::design-comuni.pages.'.$slug)

{{-- MAI usare namespace tema-specifico --}}
<x-sixteen::blocks.navigation.header-main>

{{-- ✅ CORRETTO: Namespace pub_theme --}}
<x-pub_theme::blocks.navigation.header-main>
```

**Perché:**
- ✅ **DRY:** Header/footer scritti UNA volta in `app.blade.php`
- ✅ **KISS:** Page template gestisce SOLO routing e contenuto
- ✅ **Manutenibilità:** Modifica header/footer in un solo file

## 7. HTML Body Identity

### Correzioni Applicate

#### 1. Header Wrapper ✅

**Prima:**
```blade
<x-section slug="header" />
```

**Dopo:**
```blade
<header class="it-header-wrapper">
    <x-section slug="header" />
</header>
```

#### 2. Main Container ✅

**Prima:**
```blade
<main id="main-container">
    @yield('content')
</main>
```

**Dopo:**
```blade
<main id="main-container" class="it-main-container">
    <div class="container">
        @yield('content')
    </div>
</main>
```

#### 3. Footer ✅

**Stato:** Già corretto
```blade
<footer class="it-footer-main-wrapper" id="footer">
    <div class="container">
        ...
    </div>
</footer>
```

**Template Analisi:**

```markdown
# Analisi: [Nome Pagina]

## Differenze Rilevate

### Header
- ❌ Logo non visibile
- ❌ Colori diversi (#0066CC vs #003366)
- ❌ Spaziatura: 16px vs 24px

### Footer
- ❌ Colonne non allineate
- ❌ Font-size: 14px vs 16px

## Come Correggere

1. Aggiornare `style-apply.css`:
   ```css

   ```bash
   npm run build
   npm run copy
   ```

3. Verificare con screenshot
```

## 11. Regole Assolute

### 🚫 MAI

```blade
❌ File specifici: homepage.blade.php, amministrazione.blade.php
❌ Namespace tema: <x-sixteen::*>
❌ Bootstrap import: @import url('bootstrap-italia.min.css')
❌ Bootstrap JS: import "bootstrap-italia/dist/js/bootstrap-italia.bundle.min.js"
❌ @include per pagine: @includeIf('pub_theme::design-comuni.pages.'.$slug)
❌ Header/footer in [slug].blade.php
❌ rm -rf storage/framework/views/* (cancella .gitignore!)
```

### ✅ SEMPRE

```blade
✅ UNO [slug].blade.php per TUTTE le pagine
✅ Namespace pub_theme: <x-pub_theme::*>
✅ Tailwind @apply (NON Bootstrap import)
✅ Alpine.js per interattività (NON Bootstrap JS)
✅ <x-layouts.app> (NON <x-pub_theme::layouts.*>)
✅ <x-section slug="header" /> e <x-section slug="footer" />
✅ JSON per contenuti: config/local/fixcity/database/content/pages/*.json
✅ Vite con secondo parametro: @vite([...], 'themes/Sixteen')
✅ Cache pulita: php artisan optimize:clear
```

### Git Workflow

```bash
# ✅ CORRETTO: Forward-only
git add .
git commit -m "feat: replicate homepage design comuni"
git push

# ❌ SBAGLIATO: MAI revert/revert
git reset --hard HEAD~1  ← MAI!
git revert <commit>       ← MAI!
```

**Perché:**
- ✅ **Git forward-only:** Solo in avanti, mai indietro
- ✅ **Documentazione:** Ogni modifica documentata

## 📝 Note Finali

### La Filosofia, La Visione, Lo Zen

> **DRY (Don't Repeat Yourself):**
> - Unico `[slug].blade.php` per tutte le pagine
> - Blocchi universali, NON pagina-specifici
> - Header/footer scritti UNA volta sola

> **KISS (Keep It Simple, Stupid):**
> - Ogni file fa UNA cosa sola
> - Logica minimale, zero complessità
> - Documentazione chiara e accessibile

> **Separation of Concerns:**
> - Layout: Struttura comune (header, footer)
> - Page Template: Routing e contenuto specifico
> - Components: Blocchi riutilizzabili

> **Git Forward-Only:**
> - Solo in avanti, mai indietro
> - Studiare versioni vecchie per capire il perché
> - Migliorare, non ripristinare

## 12. Canonical Page Tree

### Solo directory ammesse in `resources/views/pages`

```text
resources/views/pages/
├── [container0]/
│   └── [slug0]/
├── auth/
└── tests/
```

### Directory vietate

Non creare o mantenere alberi semantici hardcoded come `administration`, `news`, `profile`, `services`, `turismo` o `pages`.

### Regola operativa

- il tema usa Folio dinamico, non cartelle dedicate per dominio editoriale
