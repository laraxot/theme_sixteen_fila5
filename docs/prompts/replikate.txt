## 🔁 WORKFLOW (LOOP UNICO)

Per ogni `<pagina>`:

### 1. ANALISI
Confronta:
- Reference HTML
- Local HTML (View Source)
- Ignora script
- Focalizzati su:
  - struttura DOM
  - gerarchia
  - classi

### 2. SCREENSHOT
Fai 3 screenshot:
- originale
- locale
- confronto

Salva in:
```
docs/design-comuni/screenshots/<pagina>/
```

### 3. DIFF HTML

Documenta SOLO differenze strutturali:

Esempio:
- header wrapper mancante
- container annidato diverso
- classi mancanti (it-*)
- ordine nodi diverso

👉 **NON parlare di colori o stile qui**

### 4. FIX HTML (PRIMA DI TUTTO)
Modifica SOLO:
- blade
- componenti blocchi

Obiettivo:
- ✔ stessa struttura DOM
- ✔ stessi wrapper
- ✔ stessi livelli

### 5. FIX CSS/JS (DOPO HTML OK)

Lavora SOLO in:
- Tailwind (@apply)
- Alpine.js

Replica:
- colori
- spacing
- layout
- interazioni

### 6. BUILD
```bash
cd laravel/Themes/Sixteen
npm run build
npm run copy
php artisan optimize:clear
```

## ⚡ AUTOMAZIONE

### Script di Confronto HTML — AGNOSTICO (PRINCIPALE)

**Percorso**: `bashscripts/html/compare-html.sh`
**Docs**: [bashscripts/docs/html/compare-html.md](../../../../bashscripts/docs/html/compare-html.md)

```bash
# Confronto struttura body tra reference e locale
bash bashscripts/html/compare-html.sh \
  "https://italia.github.io/design-comuni-pagine-statiche/sito/<pagina>.html" \
  "http://127.0.0.1:8001/it/tests/<pagina>" \
  "laravel/Themes/Sixteen/docs/design-comuni/screenshots/<pagina>"
```

Genera: struttura DOM, diff unificato, metriche JSON, report Markdown.

**Output chiave** (agnostico da framework CSS):
```
--- ANALISI SEMANTICA (agnostica da CSS framework) ---
  ID trovati URL1:      calendario, footer, head-section, ...
  ID trovati URL2:      nessuno  ← indica quali ID semantici mancano
  ID mancanti in URL2:  calendario, footer, head-section, ...
```

**Interpretazione**:
- `ID mancanti in URL2` = blocchi Blade che devono ricevere gli `id` corretti
- Similitudine CSS bassa (< 10%) è normale (Bootstrap Italia vs Tailwind)
- Focus su: tutti e 6 gli ID semantici presenti → HTML strutturalmente corretto

### Script di Confronto Visuale
Disponibili in `bashscripts/design-comuni/visual-compare/`:

| Script | Scopo |
|--------|-------|
| `compare-html.js` | Confronta DOM structure (esclude scripts) |
| `compare.js` | Screenshot + pixel diff comparison |
| `run-comparison.sh` | Runner che esegue entrambi |

### Usage
```bash
# Confronto completo di una pagina
bash bashscripts/design-comuni/visual-compare/run-comparison.sh --page homepage

# Solo HTML structure
node bashscripts/design-comuni/visual-compare/compare-html.js --page homepage

# Solo visual diff
node bashscripts/design-comuni/visual-compare/compare.js --page homepage

# Tutte le pagine
bash bashscripts/design-comuni/visual-compare/run-comparison.sh --all
```

### Output
Ogni pagina genera:
```
docs/design-comuni/screenshots/<pagina>/
├── originale.png              ← Reference screenshot
├── replica.png                ← Local screenshot
├── diff.png                   ← Pixel diff
├── analisi.md                 ← Visual analysis report
├── reference-structure.txt    ← Reference DOM
├── local-structure.txt        ← Local DOM
└── html-structure.md          ← HTML comparison report
```

### Metrics
- **HTML Structure Match ≥90%**: ✅ PASS
- **70-89%**: ⚠️ WARNING
- **<70%**: ❌ FAIL
